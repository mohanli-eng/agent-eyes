# 生码约束

本文档定义所有代码生成阶段（/plan、/implement、修复方案）必须遵守的约束。每次生成或修改代码前，先对照这些约束进行方案设计。

---

## 一、环境约束

### 已有项目

- 沿用已有的语言版本、包管理器、构建工具、lint/formatter 配置
- 不引入与现有技术栈冲突的依赖
- 遵循已有的目录结构与文件命名惯例
- 遵循已有的编码风格（缩进、命名、注释语言等）

### 新项目

- 根据用户要求、语义理解与业务调研自主设计技术选型
- 选型同时满足以下五项标准：
  1. **性能良好**：适合业务规模，无明显瓶颈
  2. **安全可靠**：无已知高危漏洞，依赖维护活跃
  3. **使用门槛低**：免费或有充足免费额度，普通用户可直接使用
  4. **工程化完善**：具备构建、类型检查、lint、环境隔离能力
  5. **可维护性高**：社区成熟，文档完善，易于团队上手
- 在 Coding Plan 中说明选型理由

---

## 二、代码复用

按优先级从高到低：

1. **优先复用已有代码**：项目中已有的组件、函数、工具类能满足需求的，直接使用
2. **其次使用稳定三方包**：选择维护活跃、API 稳定的库；避免引入仅用一次的大依赖
3. **最后自行实现**：确认前两者均无法满足时，才编写新实现

---

## 三、增量升级

- 不破坏已有业务逻辑，除非需求明确要求变更
- 所有改动以"最小侵入"为原则：只改动需要改动的部分
- 避免在修复或新增功能时顺手"重构"不相关代码
- 数据结构变更须兼容已有数据格式，或提供迁移方案

---

## 四、代码质量

- **无重复代码**：相同逻辑出现两次以上，提取为函数/组件
- **合理抽象**：复杂逻辑拆分为职责单一的函数；避免过度设计（不为假设的未来需求抽象）
- **边界处理**：空值、极值、并发、网络失败等场景须有明确处理逻辑
- **错误不外漏**：所有用户可触发的错误路径必须有业务层处理，不允许未捕获的异常暴露给用户

---

## 五、可维护性

在以下位置添加注释：

- 业务流程的关键决策节点（说明"为什么"而非"做什么"）
- 复杂算法或非直觉的实现方式
- 语义不明的配置项、魔法数字、特殊约定
- 与外部系统交互的接口约定

注释语言与项目保持一致。不对自明代码加注释。

---

## 六、修复原则

- **正面解决问题**：定位根本原因，修复根本原因
- **不得阉割功能**：不允许通过删除功能、跳过逻辑、注释掉代码来让编译或测试通过
- **不引入新问题**：修复方案不得破坏已有通过的测试用例
- **遇到逻辑冲突**：停止修复，向用户说明冲突所在，等待确认后继续

---

## 七、RAG / Evaluation 工程约束

### 评估集（Golden Set）生成约束

1. **数量底线**：PRD 写"N-M 题"的，必须达到下限 N。不能少。
2. **不能凭空生成 expected_chunks**：必须通过真实数据库 lookup 获取 UUID，不能用 placeholder（如 chunk_id_N）。
3. **不能造假 ID**：若数据库还未有数据时无法 lookup，必须在交付报告中显式标注"expected_chunks 待人工 lookup"，禁止生成假 UUID。
4. **expected_answer_keywords 必须从源文档抽取**：不能凭 LLM 通用知识猜测，必须 grep 源 PDF 内容确认。

### 评估指标实现约束

1. **指标定义不能简化**：PRD 写"LLM-as-judge"的不能换成"keyword overlap"；写"Recall@K"的必须按定义算（top-K 中至少一个 expected_chunk 命中），不能换成其他公式。
2. **简化必须显式声明**：若实现因技术限制（如 Next.js context 缺失）无法按 PRD 定义实现，必须在交付报告中：
   - 标明实际实现 vs PRD 定义的差异
   - 说明为什么简化
   - 列出"完整实现需要的工作"作为 known issue
3. **禁止暗自简化**：不能把 "keyword overlap" 命名为 "Faithfulness"——这是命名误导，等同于数据虚假陈述。

### 数据污染禁令

1. **禁止 target leakage**：金标准集的 expected_chunks 不能根据当前 retrieval 系统的输出反向标注。expected_chunks 应基于"内容是否真的回答了 question"的人工判断，独立于 retrieval 实现。
2. **禁止 self-fulfilling metrics**：禁止用"系统当前输出"作为"系统应该输出"的标准。
